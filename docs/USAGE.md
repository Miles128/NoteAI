## 快速开始

### 第一步：配置API

在使用任何AI功能之前，您需要配置API：

1. 打开NoteAI应用
2. 点击顶部的"API配置"标签
3. 输入您的API Key（从OpenAI或其他提供商获取）
4. 根据需要调整其他设置
5. 点击"保存配置"

### 第二步：开始使用

根据您的需求选择相应的功能模块：

- **收集网络文章** → 使用"网页下载"功能
- **转换现有文档** → 使用"文件转换"功能
- **整理笔记** → 使用"笔记整合"功能

## 功能详解

### 1. 网页下载

#### 功能说明
将网络上的文章批量下载并转换为Markdown格式，支持自动清理和AI优化。

#### 使用步骤

1. **输入URL**
   - 在文本框中粘贴要下载的文章链接
   - 每行一个URL
   - 支持HTTP和HTTPS链接

2. **设置保存路径**
   - 点击"浏览"选择保存位置
   - 默认为用户目录下的NoteAI/notes文件夹

3. **选择AI优化**（推荐）
   - 勾选"使用AI优化转换"
   - AI会自动清理乱码、优化排版
   - 需要有效的API配置

4. **开始下载**
   - 点击"开始下载"按钮
   - 等待进度完成
   - 查看结果提示

#### 输出结果
- 每个URL会生成一个独立的Markdown文件
- 文件名基于文章标题
- 文件保存在指定目录

### 2. 文件转换

#### 支持的格式
- **PDF** (.pdf) → Markdown
- **Word** (.docx) → Markdown
- **PowerPoint** (.pptx) → Markdown
- **纯文本** (.txt) → Markdown

#### 使用步骤

1. **添加文件**
   - 点击"添加文件"选择单个或多个文件
   - 或点击"添加文件夹"批量添加整个目录

2. **设置输出路径**
   - 选择转换后文件的保存位置

3. **开始转换**
   - 点击"开始转换"
   - 等待处理完成

#### 转换特点
- 自动提取文本内容
- 保留文档结构（标题、列表、表格）
- AI智能优化排版
- 自动移除图片

### 3. 笔记整合

#### 三种整合策略

##### 3.1 机器学习分类整合

**适用场景**：
- 有大量文档需要按主题分类
- 希望自动识别文档主题
- 需要生成结构化的主题文档

**使用方法**：

1. 选择"机器学习分类整合"
2. 选择主题提取方式：
   - **自动提取**：AI自动分析并提取主题
   - **手动指定**：在文本框中输入主题列表（每行一个）
3. 设置输出路径
4. 点击"开始整合"

**输出结果**：
- 每个主题生成一个Markdown文件
- 包含该主题下的所有相关内容
- 内容经过AI重写优化

##### 3.2 RAG增强整合

**适用场景**：
- 有特定问题需要回答
- 需要从文档中检索相关信息
- 希望基于检索结果生成内容

**使用方法**：

1. 选择"RAG增强整合"
2. 在"RAG查询"框中输入您的问题或主题
3. 设置输出路径
4. 点击"开始整合"

**工作原理**：
1. 构建文档向量索引
2. 基于查询检索相关文档片段
3. 使用AI生成整合内容
4. 标注信息来源

##### 3.3 长上下文直接整合

**适用场景**：
- 文档数量适中（建议10-20个）
- 希望一次性理解所有内容
- 需要生成综合性文档

**使用方法**：

1. 选择"长上下文直接整合"
2. 设置输出路径
3. 点击"开始整合"

**特点**：
- 直接将所有文档输入AI
- 生成完整的知识框架
- 包含概念索引和总结

## 高级配置

### API配置选项

#### Temperature（温度）
- **范围**：0.0 - 2.0
- **默认值**：0.7
- **说明**：
  - 较低值（0.1-0.3）：输出更确定、更保守
  - 较高值（0.8-1.0）：输出更创造性、更多样
  - 建议：内容整理用0.3-0.5，创意写作用0.7-1.0

#### Max Tokens（最大令牌数）
- **默认值**：4000
- **说明**：控制AI输出的最大长度
- 根据文档长度适当调整

### 文件路径配置

应用使用以下默认路径：

```
用户目录/
└── NoteAI/
    ├── config.json      # 配置文件
    ├── notes/           # 默认笔记保存位置
    ├── temp/            # 临时文件
    ├── logs/            # 日志文件
    └── vector_db/       # 向量数据库
```

您可以在代码中修改 `config/settings.py` 来更改这些路径。

## 最佳实践

### 1. 网页下载

**建议**：
- 一次下载5-10篇文章，避免请求过快
- 对于需要登录的网站，可能需要额外配置
- 下载后检查内容完整性

**优化提示**：
- 使用AI优化可以获得更好的Markdown格式
- 对于代码类文章，AI会保留代码块格式

### 2. 文件转换

**建议**：
- PDF转换前确保文档可搜索（不是扫描件）
- 复杂表格可能需要手动调整
- 批量转换时建议先测试单个文件

**格式优化**：
- 转换后的Markdown可能需要微调
- 可以使用笔记整合功能进一步优化

### 3. 笔记整合

**策略选择建议**：

| 文档数量 | 推荐策略 | 说明 |
|---------|---------|------|
| 5-20 | 长上下文 | 一次性处理，生成综合文档 |
| 20-100 | 机器学习分类 | 自动分类，生成主题文档 |
| 100+ | RAG | 基于查询检索，避免上下文过长 |

**主题设置建议**：
- 自动提取适合探索性整理
- 手动指定适合有明确分类需求的场景
- 主题名称应该简洁明了

### 4. 工作流程建议

**知识收集工作流**：
1. 使用"网页下载"收集文章
2. 使用"文件转换"转换现有文档
3. 使用"笔记整合"整理所有内容
4. 定期使用RAG查询回顾知识

**学习笔记工作流**：
1. 将学习资料转换为Markdown
2. 使用机器学习分类按主题整理
3. 生成主题总结文档
4. 使用RAG进行复习问答

## 故障排除

### 常见问题及解决方案

#### Q: 应用无法启动
**可能原因**：
- Python版本过低
- 依赖包未安装

**解决方案**：
```bash
# 检查Python版本
python --version  # 需要3.8+

# 重新安装依赖
pip install -r requirements.txt --upgrade
```

#### Q: AI功能返回错误
**可能原因**：
- API Key无效
- 网络连接问题
- API额度不足

**解决方案**：
1. 检查API Key是否正确
2. 测试网络连接
3. 查看API提供商的额度情况
4. 查看日志获取详细错误信息

#### Q: 网页下载失败
**可能原因**：
- URL无效
- 网站反爬虫
- 网络超时

**解决方案**：
1. 确认URL可以正常访问
2. 尝试使用浏览器访问该URL
3. 检查是否需要特殊请求头
4. 稍后重试

#### Q: 文件转换后格式混乱
**可能原因**：
- 原文件格式复杂
- 编码问题
- 特殊元素无法识别

**解决方案**：
1. 检查原文件是否正常
2. 尝试其他转换工具预处理
3. 手动调整转换后的Markdown

#### Q: 笔记整合效果不理想
**可能原因**：
- 文档质量参差不齐
- 主题设置不当
- 文档数量过多或过少

**解决方案**：
1. 预处理文档，确保质量
2. 调整主题设置
3. 尝试不同的整合策略
4. 分批处理大量文档

### 日志诊断

查看日志是解决问题的最佳方式：

1. 在应用内点击"日志"标签
2. 查看最近的错误信息
3. 或打开日志文件：`%USERPROFILE%\NoteAI\logs\noteai_YYYYMMDD.log`

### 联系支持

如果问题无法解决：
1. 收集相关日志
2. 记录复现步骤
3. 提交Issue描述问题

## 提示与技巧

### 提高效率

1. **批量处理**：尽可能批量处理文件，减少重复操作
2. **预设配置**：在API配置中使用合适的temperature值
3. **定期整理**：定期使用笔记整合功能整理知识库

### 质量保证

1. **检查结果**：AI处理后的内容建议人工检查
2. **备份原始文件**：转换前备份重要文档
3. **版本控制**：对重要笔记使用Git等版本控制

### 安全建议

1. **保护API Key**：不要分享或公开您的API Key
2. **敏感内容**：处理敏感内容时注意数据安全
3. **定期清理**：定期清理临时文件和日志
