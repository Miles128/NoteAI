# NoteAI 工作区 Schema — 四海

> 本文件是**本工作区**的知识库宪法：目录怎么摆、主题怎么分、AI 能改什么不能改什么。  
> 与跨工作区的 `NoteAI/profile.md`（你是谁、回答风格）分工：画像管「对人」，Schema 管「对库」。

复制到工作区根目录并命名为 **`schema.md`**。

---

## 1. 工作区定位

个人 **AI 知识编译器**（不是纯笔记本、也不是一次性 RAG）：

- **采集**：网页、PDF/DOCX/PPTX 等进入工作区，能自动转 Markdown 的不要手点转换。
- **整理**：按主题文件夹 + frontmatter 归类；拿不准的进待办，不硬塞。
- **编译**：同主题多篇笔记 → `wiki/` 综述；索引与文件夹始终一致。
- **检索**：RAG助手基于本库 RAG 问答，优先引用已有笔记与综述。

侧重领域：**AI 产品、AI Agent 架构、RAG/知识库、大模型应用、效率工具**。  
表述习惯：**中文优先、简洁技术向、少空泛套话**。

---

## 2. 目录结构（三层 + 运行时）

| 路径 | 角色 | 说明 |
|------|------|------|
| `Notes/` | **源稿层** | 原始 Markdown；按主题分文件夹；人类主编辑区 |
| `wiki/` | **编译层** | `WIKI.md` 索引、`{叶主题名}_综述.md`、`log.md` 变更日志 |
| `Raw/` | **原件归档** | PDF/DOCX 等；转换后笔记在 `Notes/`，原件留此处 |
| `.noteai/` | 运行时 | RAG 索引、`memory/`、操作日志、入库状态 |
| `.ai_memory/` | 工作区记忆 | `project_rules.md`（本库领域细则）、`user_profile.json` |
| 根目录 `schema.md` | 本文件 | 结构与 AI 边界 |
| `.pending_topics.json` | 待办 | 待确认主题分类 |
| `.links.json` | 待办 | 待确认双向链接 |

**不要**在 `Notes/`、`wiki/`、`Raw/` 下再建 `ai` / `wiki` 等别名目录（会被监视器忽略）。

## 3. 主题体系（最多三级）

### 3.1 表示方式

- **逻辑路径**：`一级 > 二级 > 三级`（分隔符两侧各有一个空格）
- **文件夹**：`Notes/一级/二级/三级/文章标题.md`
- **frontmatter**：与文件夹一致，例如 `topic: AI Agent > 记忆系统 > 长期记忆`

### 3.2 层级约定

| 层级 | 含义 | 示例（可按你库内实际改名） |
|------|------|---------------------------|
| 一级 | 知识领域 | `AI Agent`、`RAG 与检索`、`AI 产品`、`工具与效率` |
| 二级 | 方向/子领域 | `记忆系统`、`架构设计`、`面试与职业` |
| 三级 | 具体子题（可选） | `长期记忆`、`MCP 协议` |

- **最多三层**；三级目录下不再建「主题子文件夹」，多篇文章直接放同一三级目录。
- 单篇可直接放在二级目录下（无三级时）：`Notes/AI Agent/记忆系统/某文.md`，`topic` 写两级即可。

### 3.3 事实来源（重要）

1. **`Notes/` 文件夹路径是主题的唯一事实来源。**
2. `wiki/WIKI.md` 必须与 `Notes/` 文件夹结构 **完全一致**；应用内增删主题、移动/删除笔记后，会按文件夹 **重建** WIKI。
3. frontmatter 的 `topic` 若与所在文件夹不一致，**以文件夹为准** 并回写 frontmatter。
4. 不确定归哪一类 → 写入 **待办**，不要自动扔进「其他 / 杂项 / 未分类」。

### 3.4 主题命名

- **中文优先**；与文件名、文章标题一致或接近。
- 2～8 个字为宜；避免过大桶：`其他`、`杂项`、`资料`、`未分类`。
- 新建主题前 **优先复用** WIKI / 已有文件夹里出现过的主题名。
- 更细的领域规则（允许哪些一级、禁止合并哪些方向）写在 `.ai_memory/project_rules.md`。

## 4. 文章与 Frontmatter

### 4.1 文件命名

- 文件名 ≈ 文章标题（stem 即标题）
- 禁用字符：`/ \ : * ? " < > |`
- 采集/转换后尽量保持中文名，不批量改成拼音或 UUID

### 4.2 推荐 frontmatter

```yaml
---
topic: AI Agent > 记忆系统
tags: [Agent, 记忆, RAG]
title: 文章标题
source: 网页 | 转换 | 手写
date: 2026-05-20
---
```

| 字段 | 要求 |
|------|------|
| `topic` | 必填（归类完成后）；与 `Notes/` 路径一致 |
| `tags` | 2～5 个中文或有区分度的词；不要「学习」「笔记」「资料」等泛词 |
| `title` | 与显示标题一致 |
| `source` | 可选，便于追溯 |

### 4.3 标签规则

- 从标题 + 正文提取；**中文优先**
- 要有分类意义；同义只保留一个
- 自动打标后可在 UI 中改，AI 不批量覆盖你已改好的 tags

## 5. 采集与入库

### 5.1 来源

| 来源 | 落点 |
|------|------|
| 网页下载 | 工作区 → 再归类到 `Notes/` |
| 导入文件 | 先复制到 `Raw/`，再转 MD → `Notes/` |
| 工作区内 PDF/DOCX 等 | 自动转换（**跳过已在 `Raw/` 的原件**） |

### 5.2 入库流水线（Ingest）

顺序：**schema.md → 转换 → 分类 → 索引 → 级联综述 → 同步 WIKI**

- 打开工作区或监视到变更时，可自动跑增量流水线。
- 分类失败的进待办；级联只处理**本轮触达的主题**，不会全库逐主题重跑综述。

## 6. wiki 编译层

### 6.1 WIKI.md

- 仅作 **主题索引**：二级/三级标题 + 其下文件列表（`### 文件名.md`）
- **不**在 WIKI 里写长文；长文只在 `Notes/` 与综述里
- 与 `Notes/` 不同步时，以文件夹为准重建，不手工维护两套结构

### 6.2 综述 `{叶主题名}_综述.md`

- 位置：`wiki/` 下，文件名取主题路径的 **最后一级** + `_综述.md`  
  例：主题 `AI Agent > 记忆系统` → `wiki/记忆系统_综述.md`
- 内容：综合该 `topic` 下 **所有** `Notes/` 笔记（读，不改写源稿）
- 风格：结构清晰的技术综述；**括号注明来源文件名**；增量更新时 **保留** 已有有效段落
- 触发：新资料归类、确认待办主题、保存带 topic 的笔记、入库级联等

### 6.3 变更日志

- 级联 / 主题操作 / 入库 / Lint / 问答归档：`wiki/log.md`（按日 `- \`HH:MM:SS\` **TYPE** 消息`）

## 7. AI 可写范围（机器可读）

```yaml
ai_may_edit_wiki: true
ai_may_edit_notes: false
max_topic_depth: 3
```

| 允许 | 禁止 |
|------|------|
| 生成/更新 `wiki/*_综述.md` | 改写 `Notes/` 正文段落 |
| 重建 `wiki/WIKI.md` 索引 | 未经确认删除笔记或主题 |
| 写 `wiki/log.md`、级联 changelog | 创建第四级主题目录 |
| 归类时写/改 frontmatter 的 `topic`、`tags` | 把「待办」里的文章强行自动归类 |
| 增量更新 RAG 索引 | 修改 `Raw/` 原件 |

人工锁定：在需保留的段落外加 `<!-- human-lock -->`，综述更新不得删改该段。

## 8. 冲突与异常

1. **文件夹 vs frontmatter** → 以文件夹为准。  
2. **WIKI vs 文件夹** → 以 `Notes/` 为准，同步重建 WIKI。  
3. **综述 vs 某篇笔记观点不一致** → 综述中可并列记录差异，并注明来源文件；不擅自改笔记正文。  
4. **链接** → 自动发现的反向链接默认 **待确认**，确认后才写入稳定链接关系。  
5. **重复主题名** → 合并前需人工确认（应用内提供合并工具）。

---

## 9. 与 Karpathy「编译」对齐的本库原则

- **Stop re-deriving, start compiling**：重复查过的资料应沉淀在 `wiki/` 综述，而不是只留在对话里。
- **Notes = 不可变源稿**（语义上）；**wiki = 可演进编译产物**。
- 一篇新笔记进来：归类 → 索引 → **只更新受影响主题的综述**，不动同主题其它笔记正文。
