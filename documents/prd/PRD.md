# NoteAI PRD — AI 原生个人知识库

**版本**：v3.0  
**日期**：2026-06-30  
**状态**：产品方向重构

---

## 1. 产品定位

**一句话**：AI 帮你整理、编译、管理知识库，你只管丢资料进去。

**目标用户**：不想费劲整理、也不想从零开始的知识工作者。

**灵感**：Karpathy LLM Wiki — Stop re-deriving, start compiling.

**与竞品的区别**：
- Obsidian + ChatGPT = 两套系统，每次提问从零开始
- NoteAI = 采集 → 整理 → 编译 → 操作，一站式闭环

---

## 2. 核心循环

```
丢资料 → AI 整理(分类/标签/链接) → AI 编译(综述/WIKI) → CLI agent 操作
```

三大支柱：

| 支柱 | 职责 |
|---|---|
| **整理** | 自动分类、标签提取、双向链接发现 |
| **编译** | 综述生成/级联更新（阅读层 + RAG 索引层双重用途） |
| **CLI Agent** | 执行文件操作（移动、创建主题、归档） |

RAG 定位为辅助检索，不是核心支柱。

---

## 3. 关键架构决策

| 项 | 决策 |
|---|---|
| 三层架构 | 保留 Notes / wiki / Raw |
| Ingest pipeline | 默认**关**（惰性），设置可切换为主动触发 |
| 综述生成 | 批量入库结束后**后台异步**生成，UI 提示「正在生成」 |
| RAG vs CLI | 分离，不合并意图路由 |
| 对话记忆 | 不做 L1/L2 记忆体系 |
